## 4. Middleware Stack

### Execution Order (per request)
1. **CORS** — Allows CLIENT_URL and ADMIN_URL origins
2. **Body parsers** — JSON (10mb limit), URL-encoded, cookies
3. **Static files** — `/uploads` served from disk
4. **Morgan** — Request logging (dev mode only)
5. **Routes** — `/api/v1` → route handlers → controllers
6. **404 handler** — Unknown routes
7. **Error handler** — Global error handling

### Auth Middleware (`auth.middleware.js`)
- **`protect`** — Checks Bearer token, verifies JWT, attaches `req.user`
- Returns 401 if: no token, invalid token, expired token, user not found

### Rate Limiters (`rateLimiter.middleware.js`)
- **`authLimiter`** — 20 requests per 15 min per IP (login/register)
- **`pageviewLimiter`** — 60 requests per min per IP (pageview tracking)

### Upload Middleware (`upload.middleware.js`)
- Uses **disk storage** if Cloudinary not configured (dev mode)
- Uses **memory storage** if Cloudinary is configured (production)
- Accepted: all common image MIME types
- Max file size: 50MB

### Validation Middleware (`validate.middleware.js`)
- Takes a Zod schema, validates `req.body`
- Returns 400 with field-level error messages on failure

### Error Handler (`error.middleware.js`)
Handles:
- Multer errors (file size, count, unexpected field)
- Mongoose CastError (bad ObjectId)
- Mongoose duplicate key (11000)
- Mongoose ValidationError
- JWT errors (invalid, expired)
